Introduction

In this lab you will implement the inverse kinematics of a bicycle-model robot. Given a desired linear velocity and angular velocity, you will compute the wheel speed and steering angle that the robot's motors must produce to achieve that motion — then verify your result against an animated trajectory simulation.

The bicycle model is one of the most widely used kinematic models in mobile robotics and autonomous vehicles. Mastering its inverse kinematics is a prerequisite for path-following, waypoint navigation, and model-predictive control.

Background

The bicycle model simplifies a four-wheeled vehicle into two axles: a fixed rear drive axle and a steerable front axle. The distance between the axles is called the wheelbase, denoted L.

      δ (steering angle)
     /
[front wheel]
     |
     |  L  (wheelbase)
     |
[rear wheel] ──▶ heading θ

The robot accepts two high-level command inputs:

  • v_cmd — desired linear velocity (m/s): how fast the robot should move forward
  • ω_cmd — desired angular velocity (rad/s): how fast the robot's heading should rotate

Your task is to translate these into two low-level motor outputs:

  • speed — rear drive wheel speed (m/s), set via setWheelSpeed(speed)
  • δ — front steering angle (radians), set via setSteeringAngle(delta)

The relationship comes from the no-slip kinematic constraint. The robot's heading rate is:

ω = (speed / L) × tan(δ)

Rearranging to solve for the steering angle (the inverse kinematics formula):

δ = arctan( ω_cmd × L / v_cmd )

The simulated robot in this lab has a wheelbase of ROBOT_WHEELBASE = 1.0 m. This constant is defined in the simulation engine and must be used in your implementation — do not hardcode the number 1.0.

Introduction

In this lab you will study how a Kalman filter fuses three noisy sensors into a single, statistically-optimal estimate of a robot's motion. A robot drives along a circular path while three sensors report on its motion every time step: a wheel encoder measures linear velocity, an accelerometer measures tangential acceleration, and a gyroscope measures rotational (yaw) velocity.

The Kalman filter code is already written for you. Your job is not to implement the filter — it is to experiment with the process-noise matrix Q and the measurement-noise matrix R and observe how those two knobs control the balance between trusting the robot's motion model and trusting the noisy sensors.

Sensor fusion with a Kalman filter is one of the most important techniques in mobile robotics, self-driving cars, drones, and inertial navigation. Understanding the roles of Q and R is the key to tuning any real estimator.

Background

The motion state

The Kalman filter maintains a belief about the robot's motion state, a vector of three quantities:

      | v |     linear velocity      (m/s)     -- measured by the encoder
  x = | a |     linear acceleration  (m/s^2)   -- measured by the accelerometer
      | w |     rotational velocity  (rad/s)   -- measured by the gyroscope

The predict step (dynamics, no control input)

Every iteration the filter first predicts the next state using the robot's dynamics. There is no control input — the filter relies only on the physics model:

  v_next = v + a * dt      (velocity integrates acceleration)
  a_next = a               (acceleration assumed steady)
  w_next = w               (turn rate assumed steady)

In matrix form xk+1 = F xk with:

      | 1  dt  0 |
  F = | 0   1  0 |
      | 0   0  1 |

Because the real robot changes its acceleration and turn rate when it speeds up and slows down, the constant-a / constant-w assumption is imperfect. The process-noise matrix Q tells the filter how much to distrust this model.

The update step (measurement)

Next the filter updates the prediction using the three sensor readings. Because each sensor observes a state variable directly, the measurement matrix H is the identity. The measurement-noise matrix R encodes how noisy each sensor is.

The two knobs you will tune

MatrixMeaningSmall valueLarge value
R (measurement noise) How much to distrust the sensors Trust the sensors — estimate hugs the noisy readings Distrust the sensors — estimate leans on the model, stays smooth but can lag
Q (process noise) How much to distrust the model Trust the model — smooth, slow to react to change Distrust the model — reacts quickly to the sensors, but noisier

A sensible starting point sets R to the true sensor variances (the squares of the sensor noise standard deviations defined in the notebook).

Reading the Gaussian belief plots

The dashboard draws, for each variable, three bell curves at the current step:

  • Orange — prediction (prior): the belief after the predict step, before seeing the sensor.
  • Green — measurement: the sensor reading and its noise.
  • Blue — updated (posterior): the fused belief after the update step.

The posterior is always narrower than both inputs — that sharpening is exactly what "fusion" means: combining two uncertain sources yields a more certain estimate.

Introduction

In this lab you will implement a full-step drive sequence for a two-phase stepper motor. You will write Python code that produces the correct coil energization states to rotate the rotor consistently in one direction, then verify your sequence using an animated motor simulation.

Stepper motors are used throughout robotics — in 3D printers, CNC machines, camera gimbals, and precision actuators — because they move in discrete, repeatable steps without requiring position feedback. Understanding how to generate a correct drive sequence is the first step toward controlling them reliably.

Background

The simulated stepper motor is a 2-phase model with four electromagnetic coil positions arranged in a cross pattern around a permanent-magnet rotor:

         [Top coil — Phase A]
              |
[Left coil]--( )--[Right coil]
   Phase B         Phase B
              |
        [Bottom coil — Phase A]

Each coil position can be driven North (+1), South (−1), or Off (0). A state is expressed as a tuple (A, B) where:

  • A controls the top/bottom coil pair (Phase A)
  • B controls the right/left coil pair (Phase B)
  • +1 → first coil in the pair is North, opposite is South
  • −1 → polarity reversed
  • 0 → phase is off (no energization)

In full-step two-phase drive, both phases are energized simultaneously on every step (A ≠ 0 and B ≠ 0). The rotor snaps to the combined electromagnetic field angle at each step. The four unique states form a repeating 4-step cycle:

StepABField angleRotor angle
1+1+145°45°
2+1−1135°135°
3−1−1225°225°
4−1+1315°315°

Repeating this 4-state pattern continuously rotates the motor in one direction. Reversing the pattern order reverses the direction.
